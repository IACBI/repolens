# Governance

RepoLens currently uses maintainer-led governance.

## Maintainer

The maintainer is responsible for project direction, release readiness, issue triage, and final decisions on changes.

## Decision Making

Decisions should prioritize:

- local-first behavior
- correctness
- maintainability
- deterministic output
- practical developer workflows

## Scope Control

Features outside the MVP, such as dashboards, hosted services, cloud sync, authentication, embeddings, or default network AI calls, require explicit design discussion before implementation.
