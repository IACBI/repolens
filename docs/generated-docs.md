# Generated Docs

[English](generated-docs.md) | [Türkçe](generated-docs.tr.md)

`scan` writes five Markdown files.

## `ARCHITECTURE.md`

Summarizes project type, technologies, folder structure, entry points, runtime clues, a Mermaid module diagram, risks, unknowns, and evidence.

## `MODULES.md`

Lists detected modules with likely responsibility, important files, internal dependencies, external dependencies, confidence, and evidence.

## `ONBOARDING.md`

Provides dependency installation hints, run commands, test commands, first files to read, first contribution areas, and common package scripts.

## `DEPENDENCY_MAP.md`

Shows internal dependency edges, external package usage, Mermaid dependency diagrams, and possible coupling hotspots.

## `DOCS_HEALTH.md`

Reports documentation files found, broken file or directory references, documented package scripts that do not exist, missing recommended docs, and suggested fixes.

## Accuracy Model

Generated docs are based on static source evidence. RepoLens uses cautious language and lists evidence files because it cannot know business intent that is not visible in code.
